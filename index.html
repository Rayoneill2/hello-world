Metrics Game Plan

Requirements

  • Collect metrics for apps and infra from all AZs
  • Data Persistence
    • Do we want retention set by metric?
    • How much retention do we want over all?
  • Dashboard
    • Do we want users to be able to query and create their own dashboards?
  • Alert on trends
    • need to able to alert when an unusual trend or pattern occurs. This will need to integrate into Pager Duty?
  • Multi tenancy
    • Is it possible in graphite?
    • Need central point for data store?

Stories

  1. [SPIKE] Research: Multiple AZs data collection/collocation

    • Understand and document how graphite and logsearch could be integrated with our current AZs setup to show metrics from independent AZs and all AZs.
    • Perform and document any PoC needed.

  2. [SPIKE] Research: Data persistence and retention

    • Understand and document graphite and logsearch data persistence, specifically how metris will be persisted in our setup, and how data retention can be set.

    • Perform and document any PoC needed.

  3. [SPIKE] Research: UI for metrics

    • Understand and document how will dashing, kibana, graphana would fit in our enviroment. List capabilities of each of them. Document integration capabilities with Elasticsearch and graphite

  4. [SPIKE] Research: Alerting on metrics trends

    • Understand and document Alerts on metrics trends with ES and graphite. Also see how those alerts can be sent to pagerduty

  5. [SPIKE] Research: LDAP integraion and authorization for metrics

    • Understand and document how LDAP can be integreated with ES and graphite. Also see how metrics can be restricted to only authorized users based on role/organization
